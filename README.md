# BoSOS-Workshop

The BoS OS helps you understand and manage your company operating system.

Most companies add AI tools on top of an operating system nobody has written down. Strategy lives in the founder's head. Roles blur. Decisions don't stick. AI amplifies whatever's already there. Clarity becomes velocity; muddle becomes noise.

The Workshop skill takes the foundation Bootstrap built and makes it real.

## What this skill does

**Workshop** refines the OS that Bootstrap produced — using your insider knowledge to challenge assumptions, sharpen priorities, and turn a skeleton into something that reflects how you actually run the business.

Bootstrap produces role stubs — a first draft of who does what. Workshop turns those into full agent specs: decision boundaries, escalation triggers, evaluation criteria. That is the step where AI stops operating on public information and starts operating on how your company specifically works.

> **Bootstrap vs Workshop:** Bootstrap creates a first draft from public information, fast, external, minimal input from you. Workshop refines that draft using your insider knowledge. Run Bootstrap first, then Workshop to make it real.

## Getting started

The Workshop skill file is maintained in the canonical repo:

- **[BoS-OS-Start](https://github.com/BoSMark/BoS_OS_Start)**

Install from there. Full installation instructions are in that repo.

> **Important:** Workshop must be run inside a Cowork project with a folder selected — the same one you used for Bootstrap. If you run it from a general Claude conversation, files won't save and you'll need to start again.

---

## What comes before Workshop

Workshop requires Bootstrap to have run first. If you haven't done that:

1. Install `agent-os-bootstrap.skill` from the [BoS_OS_Start release page](https://github.com/BoSMark/BoS_OS_Start/releases/latest)
2. Run Bootstrap: type **Bootstrap my company OS** in your Cowork project
3. Once Bootstrap is complete, come back and run Workshop

Workshop is designed to pick up exactly where Bootstrap left off. It reads what Bootstrap produced and works with it.

---

## What Workshop produces

By the end of a Workshop session you will have:

- Strategy documents sharpened with your insider knowledge, not just public information
- Full agent specs for the roles Bootstrap stubbed out — each one with a decision boundary, escalation triggers, and evaluation criteria
- A first mission brief, rooted in your North Star metric

---

## What comes next

Once Workshop has hardened your OS, use the **Run** skill to operate it day to day — shaping missions, staffing them with agents, and driving delivery.
